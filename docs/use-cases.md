# DriveMind Use Cases

DriveMind v0.6 is for work that risks degrading over time.

Not every task needs it.
It matters most when the collaboration itself needs protection and future compounding.

---

## 1. Long-running project work

Use DriveMind when a project stretches across many sessions, many choices, and many opportunities for the real objective to drift.

The value here is:
- preserving the working thread
- preventing goal drift
- preserving continuity for later resumption

---

## 2. High-stakes work with boundary pressure

Use DriveMind when urgency, importance, or ambiguity could slowly erode authorization discipline.

The value here is:
- preserving boundary integrity
- asking before risky or irreversible moves
- preventing pressure from becoming fake permission

---

## 3. Difficult debugging and diagnosis

Use DriveMind when the task is blocked and there is risk of switching from diagnosis into fake motion.

The value here is:
- naming the real blocker
- choosing bounded alternatives
- restoring momentum without brute-force retries

---

## 4. Cross-session or pause-and-resume work

Use DriveMind when the work is likely to pause and resume later.

The value here is:
- preserving resume fidelity
- capturing objective, state, blocker, and next action
- reducing continuity loss across time

---

## 5. Meaningful work that should compound

Use DriveMind when finishing the task should leave behind stronger future defaults.

The value here is:
- review
- lesson
- next-time rule
- handoff residue
- SOP fragment when repetition justifies it

---

## 6. Human-AI work that should feel like real collaboration

Use DriveMind when the user wants a collaborator that can:
- keep going without going reckless
- stay visible without becoming noisy
- preserve the work thread under pressure
- make the next similar task easier, not equally costly

---

## Core rule

DriveMind is most useful when the task is not just “hard,” but vulnerable to:
- drift
- boundary loss
- continuity decay
- fake-motion recovery
- closure without residue

If none of those are in play, direct execution is usually enough.
