# Minimal Demo

## User instruction
Enable DriveMind in execution mode for this task. Do not give up too early, but stop if the risk boundary becomes unclear.

## Expected behavioral changes
### Before DriveMind
- weaker persistence
- less explicit progress structure
- less reusable post-task output

### After DriveMind
- clearer task objective restatement
- stronger evidence-driven retry behavior
- explicit boundary / escalation reporting
- short review and reusable lesson at the end

## Minimal observable output
1. Objective
2. Progress made
3. Blocker / uncertainty
4. Next action
5. Escalation boundary
6. Reusable lesson

## Why this demo matters
It shows that DriveMind changes not only what the agent says, but how it behaves across the entire task.
